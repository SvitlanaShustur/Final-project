1. SQL Queries Implementation

Метою SQL-частини проєкту було отримання та узагальнення даних із реляційної бази BigQuery для подальшої аналітичної обробки та візуалізації результатів.

Запит 1: Користувачі з Бразилії (2023)

SELECT first_name, last_name, email
FROM `bigquery-public-data.thelook_ecommerce.users`
WHERE country = 'Brasil' 
  AND created_at BETWEEN '2023-01-01' AND '2023-12-31';

Запит 2: Топ-категорії товарів

SELECT category, COUNT(id) AS quantity
FROM `bigquery-public-data.thelook_ecommerce.products`
GROUP BY category
ORDER BY quantity DESC;

Запит 3: Аналіз замовлень (JOIN)

SELECT ord.order_id, usr.first_name, usr.last_name, ord.status
FROM `bigquery-public-data.thelook_ecommerce.orders` AS ord
JOIN `bigquery-public-data.thelook_ecommerce.users` AS usr ON usr.id = ord.user_id
WHERE status = 'Shipped';

Запит 4: Найдорожчі замовлення

SELECT order_id, user_id, ROUND(SUM(sale_price), 2) AS sum_orders
FROM `bigquery-public-data.thelook_ecommerce.order_items`
GROUP BY order_id, user_id
ORDER BY sum_orders DESC
LIMIT 10;

Запит 5: Географія покупців (>500 користувачів)

SELECT COUNT(DISTINCT(id)) as user_count, country
FROM `bigquery-public-data.thelook_ecommerce.users`
GROUP BY country
HAVING user_count > 500
ORDER BY user_count DESC;

Дисконтна політика неефективна: знижки не підвищують кількість замовлень, а лише зменшують маржу

Парадокс середнього чека Середній чек у замовленнях зі знижкою нижчий, ніж у замовленнях за повну вартість

Стратегічні рекомендації:

  • Лояльність: Відмовитися від загальних дисконтів на користь бонусної системи (за прикладом мережі 'Сільпо'). Бонуси на наступний місяць стимулюватимуть клієнтів повертатися.
  • Retention: Впровадити email-маркетинг та ремаркетинг для клієнтів після першої покупки, щоб зменшити Lifetime
  • Маркетинг: Впровадити точкові знижки лише для стимуляції повторних продажів..
  • Дані: Для подальшого аналізу необхідно налагодити збір даних на рівні окремих продуктів, щоб чітко бачити товари-лідери та аутсайдери

3. Linear Regression Model

Ціль створення моделі

Метою побудови моделі лінійної регресії було визначення та аналіз факторів, що мають найбільш суттєвий вплив на суму продажів (Gross Amount). Це може дозволити зрозуміти, які характеристики клієнтів або параметри замовлень найкраще прогнозують обсяг продажів та оптимізувати маркетингові стратегії

Підготовка та інженерія ознак (Feature Engineering)

Очищення даних: Видалено технічні ідентифікатори (CID, TID) та оброблено пропуски для усунення "шуму"

Робота з часом: Дату покупки конвертовано у формат datetime, на основі чого створено змінні: рік, місяць та день тижня

Кодування категорій: Стать, категорію товару, метод покупки та локацію трансформовано за допомогою One-Hot Encoding

Обробка віку: Для груп "Age Group" створено числовий аналог

Побудова моделі та оцінка результатів

Структура моделі: Для навчання було сформовано матрицю ознак X та цільову змінну y (Gross Amount) з розподілом на навчальну та тестову вибірки - 80 на 20

Контроль витоку даних: Для забезпечення достовірності було перевірено взаємозв’язок між Gross/Net Amount та знижками, щоб уникнути штучного завищення точності (витоку даних)

Метрики якост: Ефективність моделі оцінювалася за коефіцієнтом детермінації (R²) та середньою абсолютною помилкою (MAE)

Key Takeaways (ML):

  • Стабільність: Поведінка клієнтів не залежить від віку

  • Перспектива: Потрібен аналіз на рівні конкретних товарів
